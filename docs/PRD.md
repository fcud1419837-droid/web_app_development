# 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：現代人生活忙碌，面對工作、感情、生活等多方面的壓力，常常對未來感到迷惘，需要心靈上的指引與寄託。傳統的算命、宮廟抽籤受限於時間與空間，因此開發一款線上算命系統，讓使用者能隨時隨地進行占卜、抽籤，獲得心靈上的慰藉，並提供樂捐（香油錢）功能以維持系統營運。
- **目標用戶**：對未來感到迷惘、需要做決策參考、尋求心靈寄託、或單純對命理占卜有興趣的普羅大眾（特別是習慣使用網路的年輕至中壯年族群）。
- **核心價值主張**：打破時空限制，提供便捷、有趣且具備儀式感的線上占卜體驗，並透過儲存歷史紀錄讓使用者追蹤自己的運勢變化。

## 2. 功能需求
以下列出本系統的主要功能與對應的使用者故事：

### (1) 線上算命、占卜與抽籤 (核心功能)
- 作為 **使用者**，我希望 **可以在線上進行各種形式的算命（如：宮廟抽籤、塔羅牌等）**，以便 **獲得針對我目前問題的神明指示或運勢分析**。

### (2) 儲存算命結果 (歷史紀錄)
- 作為 **使用者**，我希望 **系統能自動儲存我過去的抽籤或占卜結果**，以便 **日後回顧我的運勢變化與當時的指引**。

### (3) 捐香油錢 (贊助系統)
- 作為 **使用者**，我希望 **能夠透過線上支付或轉帳的方式捐獻「香油錢」**，以便 **對神明表達感謝，並還願支持這個系統的營運**。

### (4) 會員註冊與登入
- 作為 **訪客**，我希望 **可以註冊並登入系統**，以便 **擁有個人的專屬帳號、跨裝置同步我的測算紀錄**。

### (5) 分享算命結果
- 作為 **使用者**，我希望 **能將抽籤或占卜結果一鍵分享到社群媒體 (如 IG、Facebook、LINE)**，以便 **跟朋友分享我的好運或尋求朋友的意見**。

## 3. 非功能需求
- **技術限制**：使用 Flask + Jinja2 作為後端與樣板引擎，資料庫使用 SQLite 進行數據管理。
- **效能考量**：抽籤與查詢歷史紀錄的反應時間需小於 1 秒，確保流暢的使用者體驗；前端需具備響應式設計 (RWD)，適應手機與電腦螢幕。
- **安全考量**：使用者密碼需經過加密（如 bcrypt）儲存確保個資安全；捐香油錢的金流串接需符合當地法規與第三方支付 API 的安全規範。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**
  - 基礎會員系統（註冊/登入）
  - 核心占卜功能（單一種類，例如：線上宮廟抽籤，需包含求籤、擲筊確認、取得籤詩與解籤）
  - 儲存並查看個人歷史抽籤紀錄
- **Should Have (應該有)**
  - 捐香油錢功能介面（顯示匯款帳號或簡單的金流按鈕）
  - 籤詩結果社群分享按鈕
- **Nice to Have (加分項)**
  - 精緻的抽籤動畫（如籤筒搖晃、擲筊動畫，增加儀式感）
  - 多種占卜方式（擴充塔羅牌、每日運勢等）
  - 香油錢排行榜或點燈祈福功能
